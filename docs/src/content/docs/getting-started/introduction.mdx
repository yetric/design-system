`@yetric/ui` is a React component library built for modern applications. It covers everyday UI needs, advanced data patterns, and the streaming chat interfaces that AI-powered apps demand.

## Philosophy

**Provider-agnostic.** Components like `AIChat`, `AddressInput`, and `DataGrid` take callbacks instead of hardcoded integrations. Bring your own LLM, geocoding service, or data layer.

**Accessible by default.** Complex behaviour (focus trapping, keyboard navigation, ARIA roles) is handled by [Radix UI](https://radix-ui.com) primitives. We focus on visual design, API consistency, and documentation.

**Boring and predictable.** APIs are consistent across components. No surprise prop names, no magic.

## When to use it

`@yetric/ui` is a good fit when you:

- Want a complete component set rather than assembling pieces from multiple libraries
- Are building AI-powered features and need streaming UI primitives
- Need dark mode and theming to work out of the box
- Want accessible components without managing Radix UI yourself

## When not to use it

- You need a headless library with zero default styles
- You are already deep into a different component system (shadcn/ui, MUI, etc.)
- You need IE11 support (ESM-only build)
